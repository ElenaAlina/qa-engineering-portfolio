# Vulnerability Reporting Methodology

## 1. Verify
- Reproduce multiple times
- Ensure it is not a false positive

## 2. Document
- Affected endpoint
- Request and response
- User role
- Environment

## 3. Create Steps
- Step-by-step reproduction
- Minimal required actions

## 4. Create Proof of Concept
- Screenshots
- HTTP requests
- Payloads

## 5. Assess Impact
- Confidentiality
- Integrity
- Availability

## 6. Suggest Remediation
- Input validation
- Access control
- Output encoding

## 7. Submit
- Clear title
- Clear summary
- Concise report
